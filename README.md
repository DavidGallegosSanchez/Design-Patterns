# 🏗️ The Design Patterns Playground

Welcome to my personal catalog of software design patterns! This repository serves as a living documentation and code laboratory where I break down classic software design patterns, understand their trade-offs, and implement them in practice.

## 🗂️ What's Inside?

The patterns are categorized into the three classic pillars:

### 1. Creational Patterns
*Mechanisms for object creation that increase flexibility and reuse of existing code.*
- [ ] Factory Method
- [ ] Abstract Factory
- [ ] Singleton
- [ ] Builder
- [ ] Prototype

### 2. Structural Patterns
*How to assemble objects and classes into larger structures while keeping them flexible and efficient.*
- [ ] Adapter
- [ ] Bridge
- [ ] Composite
- [ ] Decorator
- [ ] Facade
- [ ] Proxy

### 3. Behavioral Patterns
*Effective communication and the assignment of responsibilities between objects.*
- [ ] Chain of Responsibility
- [ ] Command
- [ ] Iterator
- [ ] Observer
- [ ] Strategy
- [ ] State

## 🚀 Goals of this Repo
- Understand the **why** behind each pattern, not just the **how**.
- Identify real-world scenarios where these patterns prevent messy code.
- Keep implementations clean, readable, and well-tested.
